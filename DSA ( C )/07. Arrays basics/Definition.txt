## 7. Arrays ‚Äì Basics (DSA)

Arrays are one of the most fundamental data structures in Data Structures & Algorithms. Almost every advanced structure (strings, matrices, heaps, hash tables, graphs using adjacency matrices) is built on top of arrays, so mastering them is non-negotiable.

---

## 1. What is an Array?

An array is a collection of elements of the same data type, stored in contiguous memory locations, and accessed using an index.

üìå Key idea:

> One name ‚Üí multiple values ‚Üí fixed memory block

Example (conceptual):

```
Index:   0   1   2   3   4
Value: 10  20  30  40  50
```

---

## 2. Why Arrays Are Important

Arrays are important because they provide:

 Fast access to elements using index
 Efficient memory usage (no extra pointers like linked lists)
 Foundation for many algorithms (sorting, searching, DP)

Without arrays, concepts like binary search, sliding window, prefix sums, dynamic programming would not exist.

---

## 3. Array Declaration and Initialization (C)

### Declaration

```c
int arr[5];
```

### Initialization

```c
int arr[5] = {10, 20, 30, 40, 50};
```

### Partial Initialization

```c
int arr[5] = {1, 2};
// Remaining elements become 0
```

### Size Inference

```c
int arr[] = {5, 10, 15};  // Size = 3
```

---

## 4. Indexing in Arrays

 Array indexing starts from 0
 Last index = `size - 1`

Example:

```c
arr[0] ‚Üí first element  
arr[4] ‚Üí last element (for size 5)
```

‚ö†Ô∏è Accessing out of bounds (like `arr[5]`) causes undefined behavior

---

## 5. Memory Representation

Arrays are stored in contiguous memory locations.

If:

 `arr[0]` is at address `1000`
 `int` size = 4 bytes

Then:

```
arr[0] ‚Üí 1000
arr[1] ‚Üí 1004
arr[2] ‚Üí 1008
```

üìå This is why:

```c
arr[i] = (arr + i)
```

## 7. Basic Operations on Arrays

### 1. Traversal

Accessing each element once.

```c
for(int i = 0; i < n; i++) {
    printf("%d ", arr[i]);
}
```

‚è± Time Complexity: O(n)

---

### 2. Insertion

 At end ‚Üí O(1)
 At beginning / middle ‚Üí O(n) (shifting required)

---

### 3. Deletion

 Requires shifting elements
 Time Complexity: O(n)

---

### 4. Searching

 Linear Search ‚Üí O(n)
 Binary Search (sorted array) ‚Üí O(log n)

---

## 8. Advantages of Arrays

‚úÖ Fast random access
‚úÖ Simple and easy to use
‚úÖ Cache-friendly (due to contiguous memory)
‚úÖ Efficient for fixed-size data

---

## 9. Limitations of Arrays

‚ùå Fixed size (cannot grow dynamically in C)
‚ùå Insertion & deletion are costly
‚ùå Memory wastage if size is overestimated
‚ùå Homogeneous data only (same type)

These limitations lead to Linked Lists, Vectors, and Dynamic Arrays.

## 13. When to Use Arrays vs Others

| Scenario               | Best Choice            |
| ---------------------- | ---------------------- |
| Fixed-size data        | Array                  |
| Fast indexing          | Array                  |
| Frequent insert/delete | Linked List            |
| Dynamic resizing       | Vector / Dynamic Array |

## Summary
