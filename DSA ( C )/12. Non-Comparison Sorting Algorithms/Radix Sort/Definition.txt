

### 1. What is Radix Sort?

Radix Sort is a non-comparison sorting algorithm that sorts numbers digit by digit, instead of comparing whole values.

It processes elements starting from:

 Least Significant Digit (LSD) â†’ most common
  or
 Most Significant Digit (MSD) â†’ used in some advanced variants

Radix Sort uses another stable sorting algorithm internally, most commonly Counting Sort.

---

### 2. Why is Radix Sort Non-Comparison?

 No direct element-to-element comparison (`<`, `>`)
 Ordering is based on digit positions
 Sorting is achieved using counting and grouping

Because of this, Radix Sort can achieve linear time complexity under suitable conditions.

---

### 3. Core Idea (How Radix Sort Thinks)

Instead of sorting numbers as a whole:

1. Sort by units digit
2. Then tens digit
3. Then hundreds digit
4. Continue until the highest digit

Each pass refines the order while keeping previous ordering intact (this is why stability is critical).

### 5. Why Stability Matters in Radix Sort

Radix Sort depends on stability because:

 Each digit pass must preserve the order from previous passes

### 7. Types of Radix Sort

#### 1. LSD Radix Sort

 Starts from least significant digit
 Easier to implement
 Most commonly used

#### 2. MSD Radix Sort

 Starts from most significant digit
 More complex
 Useful for strings and variable-length keys

---

### 8. When to Use Radix Sort

Use Radix Sort when:

 Data consists of integers or fixed-length strings
 Range of digits is limited
 Stability is required
 Very large datasets need fast sorting

---
