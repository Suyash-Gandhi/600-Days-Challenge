## 9. Hashing and Immutability in Blockchain


# 1. What is Hashing?

### ➤ Definition

Hashing is the process of converting any input data into a fixed-length string using a mathematical function called a hash function.

 Input → can be anything (text, file, transaction data)
 Output → fixed-size hash (e.g., 256-bit in SHA-256)

### ➤ Example

```
Input: "Hello"
Output: 185f8db32271fe25f561a6fc938b2e264306ec304eda518007d1764826381969
```

Even a tiny change in input gives a completely different output.

---

## 2. Properties of Cryptographic Hash Functions

Blockchain uses cryptographic hash functions (like SHA-256 in Bitcoin).

### Key properties:

### 1. Deterministic

Same input → same output

```
hash("hello") = hash("hello")
```

---

### 2. Fast Computation

Hashing should be quick even for large data.

---

### 3. Pre-image Resistance

Given a hash → you cannot find the original input

👉 Important for security

---

### 4. Avalanche Effect

Small change in input → completely different hash

```
"Hello" → 185f...
"hello" → 2cf2...
```

---

### 5. Collision Resistance

It’s extremely hard to find:

```
hash(A) = hash(B)
```

---

# 3. What is Immutability?

### ➤ Definition

Immutability means data cannot be changed once it is recorded.

In blockchain:

> Once a transaction is added to a block → it cannot be altered.

---

# 4. How Hashing Creates Immutability

This is the most important concept.

### Each block contains:

 Transaction data
 Its own hash
 Previous block’s hash

---

### Structure:

```
Block 1:
  Data: A
  Hash: H1

Block 2:
  Data: B
  Prev Hash: H1
  Hash: H2

Block 3:
  Data: C
  Prev Hash: H2
  Hash: H3
```

---

### What happens if someone tampers with data?

Suppose attacker changes Block 2 data:

```
Data: B → B'
```

Now:

 Hash changes → H2 becomes H2'
 Block 3 still points to old H2 ❌
 Chain breaks ❌

---

### Result:

👉 Entire blockchain becomes invalid

---

# 5. Why It’s Practically Impossible to Hack

To successfully tamper:

1. Modify block data
2. Recalculate its hash
3. Recalculate ALL subsequent blocks
4. Gain majority network control (51% attack)

👉 This is computationally infeasible in large networks

---

# 6. Real-World Analogy

Think of blockchain like a sealed notebook with page numbers and signatures:

 Each page references the previous page
 If you change one page:

   All future pages become invalid
   Everyone notices the inconsistency

---

# 7. Role of Hashing in Blockchain

### Hashing is used for:

 Linking blocks (chain structure)
 Securing transactions
 Proof of Work (mining)
 Data integrity verification
 Digital signatures (indirectly)

---

# 8. Hashing vs Encryption (Common Confusion)

| Feature      | Hashing             | Encryption      |
| ------------ | ------------------- | --------------- |
| Reversible   | ❌ No                | ✅ Yes           |
| Purpose      | Integrity, security | Confidentiality |
| Key Required | ❌ No                | ✅ Yes           |
| Example      | SHA-256             | AES, RSA        |

---

# 9. Immutability: Benefits and Drawbacks

### ✅ Benefits

 Data integrity
 Trustless system
 Tamper-proof records
 Transparency

---

### ❌ Drawbacks

 Mistakes cannot be corrected
 Hard to delete data
 Privacy concerns
