## 31. Ethereum Virtual Machine (EVM)

The Ethereum Virtual Machine (EVM) is the core execution engine of Ethereum. It is responsible for running smart contracts and handling all computations on the network in a deterministic and secure way.

---

# 🔹 1. What is the EVM?

 The EVM is a global, decentralized computer maintained by all Ethereum nodes.
 Every node runs the EVM to:

   Execute smart contracts
   Validate transactions
   Maintain consensus

👉 Think of it as:

> “A sandboxed runtime environment where smart contracts live and execute.”

---

# 🔹 2. Key Characteristics of the EVM

### 1. Deterministic Execution

 Every node must produce the same result for the same transaction.
 Ensures consistency across the network.

👉 Example:

 If a contract transfers 1 ETH → every node must agree on balances.

---

### 2. Sandboxed Environment

 Smart contracts:

   Cannot access external systems directly
   Cannot modify system files
 Prevents malicious behavior

---

### 3. Stack-Based Architecture

 Uses a Last-In-First-Out (LIFO) stack
 Operations push and pop data from the stack

---

### 4. Gas-Based Execution

 Every operation costs gas
 Prevents:

   Infinite loops
   Network abuse

---

### 5. Turing Complete

 Can perform any computation (given enough gas)

---

# 🔹 3. How the EVM Works (Execution Flow)

### Step-by-step:

1. User sends transaction

    Example: call a smart contract function

2. Transaction enters mempool

3. Validator includes it in a block

4. EVM executes the smart contract code

    Bytecode is executed instruction by instruction

5. State changes are applied

    Account balances
    Storage updates

6. All nodes verify execution

    If valid → block is accepted

---

# 🔹 4. EVM Architecture

The EVM has several core components:

### 1. Stack

 Temporary storage (max 1024 items)
 Used for calculations

---

### 2. Memory

 Temporary and volatile
 Cleared after execution

---

### 3. Storage

 Permanent storage of smart contracts
 Stored on blockchain

---

### 4. Program Counter

 Tracks current instruction

---

### 5. Gas Counter

 Tracks remaining gas

---

# 🔹 5. EVM Bytecode

 Smart contracts are written in high-level languages like:

   Solidity
 Then compiled into EVM bytecode

👉 Example flow:

```
Solidity → Compiler → Bytecode → EVM Execution
```

### 2. Ensures Trustless Execution

 No central authority needed

# 🔹 1. What actually happens when a smart contract runs?

When someone interacts with a smart contract on Ethereum:

 A transaction is broadcast to the network
 That transaction includes:

   Contract address
   Function to call
   Input data

---

# 🔹 2. Who executes the contract?

### ✅ All validating nodes (full nodes / validators) execute it

 Every node with the EVM:

   Replays the transaction
   Executes the smart contract code
   Verifies the result

👉 Why?

Because blockchain requires consensus:

> Everyone must independently compute the same result.

---

# 🔹 3. Why not only “interested” nodes?

If only some nodes executed contracts:

 Results could differ
 Malicious nodes could cheat
 Network would lose trust

👉 So Ethereum enforces:

> “Don’t trust, verify.”

Every node verifies everything.

---

# 🔹 4. Step-by-step execution flow

1. You call a smart contract
2. Transaction enters mempool
3. A validator includes it in a block
4. That validator executes the contract
5. Block is broadcast
6. All other nodes re-execute the same contract
7. If results match → block is accepted

---

# 🔹 5. Important nuance (this is where many get confused)

### ✔️ Users don’t execute contracts

 Your wallet (e.g., MetaMask) just sends the transaction

### ✔️ Nodes execute contracts

 Full nodes + validators run the EVM

### ✔️ Light nodes

 Do not execute everything
 They rely on full nodes for verification

---

# 🔹 6. Analogy (simple but powerful)

Think of it like a math exam:

 One student (validator) solves the problem first
 All other students (nodes) recheck the solution
 Only if everyone agrees → answer is accepted
