### 9. Conditional Statements in Solidity


## 🔹 1. `if` Statement

 Executes a block of code only if a condition is true

```solidity
if (x > 10) {
    // code runs if condition is true
}
```

### Example:

```solidity
pragma solidity ^0.8.0;

contract CheckNumber {
    function check(uint x) public pure returns (string memory) {
        if (x > 10) {
            return "Greater than 10";
        }
        return "10 or less";
    }
}
```

---

## 🔹 2. `if-else` Statement

 Executes one block if condition is true, otherwise another block

```solidity
if (condition) {
    // true block
} else {
    // false block
}
```

### Example:

```solidity
function checkEven(uint x) public pure returns (string memory) {
    if (x % 2 == 0) {
        return "Even";
    } else {
        return "Odd";
    }
}
```

---

## 🔹 3. `if-else if-else` Ladder

 Used when multiple conditions need to be checked

```solidity
if (x < 10) {
    // block 1
} else if (x < 20) {
    // block 2
} else {
    // block 3
}
```

### Example:

```solidity
function grade(uint marks) public pure returns (string memory) {
    if (marks >= 90) {
        return "A";
    } else if (marks >= 75) {
        return "B";
    } else if (marks >= 50) {
        return "C";
    } else {
        return "Fail";
    }
}
```

---

## 🔹 4. Ternary Operator (`? :`)

 Short-hand for simple conditions

```solidity
condition ? value_if_true : value_if_false;
```

### Example:

```solidity
function max(uint a, uint b) public pure returns (uint) {
    return a > b ? a : b;
}
```

---

## 🔹 5. Important Notes (VERY IMPORTANT)

### ✅ 1. Conditions must return `bool`

 Solidity does not allow implicit truthy/falsy values

```solidity
if (x) { } // ❌ invalid
if (x > 0) { } // ✅ correct
```

---

### ✅ 2. Gas Optimization Tip

 Complex conditions = more gas
 Keep logic simple and efficient

---

### ✅ 3. Use with `require`, `assert`, `revert`

Conditional logic is often used with error handling:

```solidity
function withdraw(uint amount) public {
    require(amount > 0, "Amount must be greater than 0");
}
```

 `require` → input validation
 `assert` → internal errors
 `revert` → manual error trigger
