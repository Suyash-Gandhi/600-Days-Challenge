### 28. Gas Basics in Solidity (Gas Limit, Gas Price, Refunds)


# 🔹 1. What is Gas?

Gas is the unit of computation used in Ethereum.

 Every operation (storage write, computation, transfer, etc.) has a fixed gas cost
 Users pay gas using ETH
 Prevents:

   Infinite loops
   Network spam
   Resource abuse

👉 Think of gas like fuel for executing smart contracts

---

# 🔹 2. Gas Limit

### ✅ Definition:

The maximum amount of gas a user is willing to spend on a transaction.

### 📌 Key Points:

 Set by the user when sending a transaction
 Acts as a safety cap
 If gas runs out → transaction fails

### ⚠️ Important Behavior:

 If gas is too low:

   Transaction fails (Out of Gas)
   But gas used is still lost
 If gas is too high:

   Unused gas is refunded

### 🧠 Example:

```solidity
for (uint i = 0; i < 1000; i++) {
    // expensive operation
}
```

 This loop consumes gas per iteration
 If gas limit is insufficient → execution stops midway

---

# 🔹 3. Gas Price

### ✅ Definition:

The amount of ETH you pay per unit of gas

### 📌 Measured in:

 Gwei (1 ETH = 1e9 Gwei)

### 💡 Formula:

```
Transaction Fee = Gas Used × Gas Price
```

### 📌 Example:

 Gas used = 21,000
 Gas price = 20 Gwei

➡️ Fee = 21,000 × 20 Gwei = 420,000 Gwei = 0.00042 ETH

---

# 🔹 4. Modern Ethereum (EIP-1559 Model)

Gas pricing is now split into:

### 1. Base Fee

 Fixed by network (depends on congestion)
 Burned (removed from circulation)

### 2. Priority Fee (Tip)

 Paid to validators/miners
 Incentivizes faster processing

### 💡 Total:

```
Total Fee = Gas Used × (Base Fee + Tip)
```

---

# 🔹 5. Gas Refunds

### ✅ What are refunds?

Ethereum gives back gas when you free up storage or optimize execution

### 📌 Common refund cases:

 Deleting storage:

```solidity
delete myVariable;
```

 Setting storage to zero:

```solidity
myVar = 0;
```

### ⚠️ Important:

 Refund is partial, not full
 Max refund is capped (~20% of total gas used)

---

# 🔹 6. Why Gas Costs Differ

Different operations cost different gas:

| Operation                | Gas Cost    | Reason                  |
| ------------------------ | ----------- | ----------------------- |
| Simple arithmetic        | Low         | Minimal computation     |
| Storage write (`SSTORE`) | High        | Blockchain state change |
| Contract deployment      | Very high   | Code storage            |
| External calls           | Medium–High | Communication overhead  |

---

# 🔹 7. Gas Optimization (VERY IMPORTANT for interviews + real projects)

### ✅ Best Practices:

### 1. Avoid unnecessary storage writes

```solidity
// expensive
count = count + 1;

// better (still costs but minimal usage matters overall)
```

---

### 2. Use `memory` instead of `storage` when possible

```solidity
function example(uint[] memory arr) public {
```

---

### 3. Use `view` and `pure`

 They don’t cost gas when called externally

---

### 4. Short-circuit conditions

```solidity
if (a && b) // stops early if a is false
```

---

### 5. Avoid loops over large arrays

 Can exceed block gas limit → fail

---

# 🔹 8. Block Gas Limit

 Each block has a maximum gas capacity
 Limits total transactions per block

👉 If your function is too heavy → it may never execute
