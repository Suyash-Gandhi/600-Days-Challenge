
## 1. What are Functional Requirements (FR)?

Functional requirements describe what the system should do.
They define features, behaviors, and actions the system must perform.

Think of them as:

> â€œIf a user does X, the system should do Y.â€

### Simple definition

Functional requirements = system capabilities

### Examples (very concrete)

 User can register using email and password
 User can log in
 User can search products
 User can place an order
 Admin can block a user
 System sends a password reset email

### Example (real project â€“ e-commerce)

 User can add items to cart
 User can remove items from cart
 User can checkout using UPI / card

## 2. What are Non-Functional Requirements (NFR)?

Non-functional requirements describe how well the system should work.
They define quality attributes, constraints, and system behavior under load.

Think of them as:

> â€œHow fast, how secure, how reliable, how scalable?â€

### Simple definition

Non-functional requirements = system qualities

### Common categories of NFRs

 Performance
 Scalability
 Availability
 Security
 Reliability
 Maintainability
 Usability
 Consistency

---

## 3. Examples of Non-Functional Requirements

### Performance

 API response time should be < 200 ms
 Page should load within 2 seconds

### Scalability

 System should handle 1 million users
 Should support 10x traffic during sales

### Availability

 System uptime should be 99.9%
 No single point of failure

### Security

 Passwords must be hashed
 APIs must require authentication
 Data must be encrypted at rest and in transit

### Reliability

 No data loss during server crash
 Messages should not be duplicated

### Maintainability

 Code should be modular
 Easy to add new features

If you remove a non-functional requirement â†’
the feature still exists, but works poorly or fails under real conditions.
