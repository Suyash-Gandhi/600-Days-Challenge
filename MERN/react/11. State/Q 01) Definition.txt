## React State

State is a built-in React object that allows components to store and manage changing data.

When the state changes, React automatically re-renders the component so the UI reflects the new data.

---

# 1. What is State in React?

State represents data that can change over time inside a component.

Examples of state:

 Counter value
 User input in a form
 Toggle (dark/light mode)
 API data
 Cart items

Example:

A counter app stores the number inside state.

When the number changes → React updates the UI automatically.

---

# 2. Why React State is Needed

Without state, React components would only display static data.

State allows components to be interactive and dynamic.

Example situations where state is needed:

| Feature           | Why State is Needed        |
| ----------------- | -------------------------- |
| Counter           | Number increases/decreases |
| Form Input        | Text updates as user types |
| Toggle Button     | ON/OFF switching           |
| Loading Indicator | Changes when API loads     |
| Shopping Cart     | Items added or removed     |

So state allows React apps to behave like real applications instead of static pages.

---

# 3. Characteristics of React State

### 1. State is Local to a Component

Each component manages its own state.

Example:

```
<LoginForm />
<Cart />
<Counter />
```

Each component has separate state.

---

### 2. State Changes Trigger Re-render

When state changes:

```
React → re-renders the component
```

This updates the UI automatically.

Example:

```
count = 0 → UI shows 0
count = 1 → UI updates to 1
```

You never manually update the DOM in React.

---

### 3. State is Mutable (but updated through setter)

State values change using setter functions.

Example:

```
setCount(count + 1)
```

React tracks this change and re-renders.

---

### 4. State Should Not Be Modified Directly

Incorrect:

```
count = count + 1
```

Correct:

```
setCount(count + 1)
```

Direct modification will not trigger React re-render.

---

# 4. Creating State in React (useState Hook)

In functional components, state is created using the useState hook.

Example:

```
import { useState } from "react";

const Counter = () => {

  const [count, setCount] = useState(0);

  return (
    <div>
      <p>{count}</p>
      <button onClick={() => setCount(count + 1)}>
        Increase
      </button>
    </div>
  );
};
```

Explanation:

| Part        | Meaning                  |
| ----------- | ------------------------ |
| useState(0) | Initial state value      |
| count       | Current state value      |
| setCount    | Function to update state |

---

# 5. How State Works Internally

When state updates:

```
User Action
      ↓
setState / setCount()
      ↓
React updates virtual DOM
      ↓
Diffing algorithm compares DOM
      ↓
Only changed parts update
```

This is why React is efficient.

---

# 6. Types of State in React

### 1. Primitive State

Simple values.

Example:

```
const [count, setCount] = useState(0);
const [name, setName] = useState("");
const [isOpen, setIsOpen] = useState(false);
```

---

### 2. Object State

```
const [user, setUser] = useState({
  name: "Suyash",
  age: 22
});
```

Updating object state:

```
setUser({
  ...user,
  age: 23
});
```

Spread operator keeps previous values.

---

### 3. Array State

```
const [todos, setTodos] = useState([]);
```

Adding item:

```
setTodos([...todos, newTodo]);
```

---

# 7. Functional State Update

Sometimes new state depends on previous state.

React provides functional update.

Example:

```
setCount(prev => prev + 1);
```

Why use this?

Because React may batch updates.

Example:

```
setCount(count + 1)
setCount(count + 1)
```

Result may still be 1 instead of 2.

Correct approach:

```
setCount(prev => prev + 1)
setCount(prev => prev + 1)
```

Now result = 2.

---

# 8. State vs Props

This is a very common React interview question.

| Feature    | State              | Props              |
| ---------- | ------------------ | ------------------ |
| Ownership  | Inside component   | Passed from parent |
| Mutability | Can change         | Read-only          |
| Purpose    | Manage data        | Pass data to child |
| Control    | Component controls | Parent controls    |

Example:

```
<Profile name="Suyash" />
```

Here name is a prop.

But inside Profile:

```
const [age, setAge] = useState(22);
```

Age is state.

---

# 9. Rules of Using State

### Rule 1

Never update state directly.

Wrong:

```
count++
```

Correct:

```
setCount(count + 1)
```

---

### Rule 2

State updates are asynchronous

React may delay updates for performance.

---

### Rule 3

Keep state minimal

Store only necessary data.

Bad:

```
fullName = firstName + lastName
```

Better:

```
store firstName and lastName
```

Compute fullName when needed.

---

# 10. Real Project Example

Example: Dark Mode Toggle

State controls UI theme.

```
const [darkMode, setDarkMode] = useState(false);
```

Toggle:

```
<button onClick={() => setDarkMode(!darkMode)}>
Toggle Mode
</button>
```

If `darkMode = true` → dark UI
If `darkMode = false` → light UI

---

# 11. Best Practices for React State

1. Keep state small and simple
2. Avoid duplicate state
3. Use functional updates when needed
4. Use separate states for unrelated data
5. Lift state up when multiple components need it

---

# 12. Common Beginner Mistakes

### Mistake 1

Direct state mutation

```
user.name = "John"
```

Wrong.

---

### Mistake 2

Updating state incorrectly for objects

```
setUser({ age: 25 })
```

This removes other properties.

Correct:

```
setUser({ ...user, age: 25 })
```

---

### Mistake 3

Using state when not needed

Sometimes simple variables are enough.
