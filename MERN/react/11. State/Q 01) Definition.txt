
# React State — Complete Explanation

## 1. What React State Actually Is

State is a built-in React mechanism that stores data which can change over time and affect the UI.

Whenever state changes, React re-renders the component so the UI reflects the new data.

Think of it like this:

 Variables store data
 State stores data that React watches
 When that data changes → React updates the UI automatically

Example idea:

```
User clicks "Like"
Likes count increases
UI updates immediately
```

The likes count must be state, otherwise React will not update the UI.

---

## 2. Why Normal Variables Do Not Work

React does not track normal variables.

Example concept:

```
let count = 0

count++
```

The value changes in memory, but React does not know about it, so UI will not update.

React only re-renders when:

 state changes
 props change
 parent re-renders

Therefore state exists so React knows when something changed.

---

## 3. The Mental Model of State

A good mental model:

```
Component = UI + State
```

Or more precisely:

```
UI = f(state)
```

Meaning:

> The UI is a function of the state.

Example:

State:

```
isLoggedIn = false
```

UI shows:

```
Login Button
```

State changes:

```
isLoggedIn = true
```

UI changes to:

```
Welcome User
Logout Button
```

So the UI reacts to the state.

This is why it is called React.

---

# 4. How State Works Internally

Each component instance maintains its own state memory.

Example:

```
<ComponentA />
<ComponentA />
<ComponentA />
```

Each one has separate state values.

Internally React stores state in a linked list structure inside the Fiber node, but conceptually you can think of it as:

```
Component
   ├── state1
   ├── state2
   └── state3
```

When a state update happens:

1. React schedules an update
2. React re-executes the component function
3. New UI is generated
4. React compares with previous UI (Virtual DOM)
5. Only necessary changes update the real DOM

---

# 5. Declaring State in React

In modern React we use useState.

Example:

```
const [count, setCount] = useState(0)
```

Two things happen here:

```
count → current state value
setCount → function to update state
```

Initial value:

```
useState(0)
```

React stores this value internally.

---

# 6. Why We Use a Setter Function

You never modify state directly.

Wrong:

```
count = count + 1
```

Correct:

```
setCount(count + 1)
```

Reason:

React needs to:

 track the change
 schedule re-render
 update the component tree

If you modify state directly, React cannot detect it.

---

# 7. What Happens When State Updates

When we call:

```
setCount(5)
```

React does NOT immediately update the state variable.

Instead React:

1. Creates an update object
2. Adds it to an update queue
3. Schedules a re-render
4. During re-render React calculates the new state

So the process is:

```
setState → queue update → re-render → new UI
```

---

# 8. State Updates Are Asynchronous

React batches updates.

Example concept:

```
setCount(1)
setCount(2)
setCount(3)
```

React does not render three times.

Instead:

```
React batches updates
final state = 3
one re-render
```

This improves performance.

---

# 9. Functional State Updates

Sometimes the new state depends on the previous state.

Example:

```
setCount(prev => prev + 1)
```

Why this is useful:

Because React may batch updates.

Example problem:

```
setCount(count + 1)
setCount(count + 1)
```

Both may use the same old value.

But this works correctly:

```
setCount(prev => prev + 1)
setCount(prev => prev + 1)
```

Now result increases twice.

---

# 10. State Causes Re-render

Every state change triggers a component re-render.

But React optimizes it:

 Virtual DOM diffing
 Fiber reconciliation
 selective DOM updates

So only necessary DOM parts change.

---

# 11. Local vs Global State

### Local State

State inside a component.

Example:

```
input text
toggle button
modal open/close
```

Used only inside that component.

---

### Global State

Shared across components.

Example:

```
user authentication
theme
cart items
notifications
```

For global state we use tools like:

 Context API
 Redux
 Zustand
 Recoil

---

# 12. Common Types of State

### UI State

```
isModalOpen
isLoading
isDarkMode
```

---

### Form State

```
input values
validation errors
```

---

### Server State

Data from API:

```
users
products
posts
```

Often managed with tools like React Query or SWR.

---

# 13. Derived State (Important Concept)

Sometimes you should not store something in state.

Example:

Wrong:

```
state:
items
totalItems
```

Correct:

```
state:
items
```

Then compute:

```
totalItems = items.length
```

This avoids duplicate state.

Rule:

> If something can be calculated from state, do not store it in state.

---

# 14. State vs Props

| Feature            | State     | Props  |
| ------------------ | --------- | ------ |
| Owned by           | Component | Parent |
| Mutable            | Yes       | No     |
| Causes re-render   | Yes       | Yes    |
| Passed to children | Yes       | Yes    |

Example:

```
Parent → passes props
Child → manages its own state
```

---

# 15. Common State Mistakes

### 1. Mutating State

Wrong:

```
stateArray.push(item)
```

Correct:

```
setArray([...stateArray, item])
```

---

### 2. Storing Too Much State

Example wrong:

```
firstName
lastName
fullName
```

Better:

```
firstName
lastName
```

Compute:

```
fullName = firstName + lastName
```

---

### 3. Too Many Re-renders

Updating state unnecessarily causes performance issues.

Example:

Updating state inside loops or effects incorrectly.

---

# 16. State and React's Render Cycle

Render flow:

```
User Action
      ↓
setState()
      ↓
React schedules update
      ↓
Component function runs again
      ↓
Virtual DOM generated
      ↓
React diffing
      ↓
DOM updates
```

---

# 17. Key Mental Model (Very Important)

Think of React like this:

```
State = Data
Component = Function
UI = Output
```

So:

```
UI = Component(State)
```

When state changes → component runs again → UI updates.

---

# Final Summary

React state is:

 a component's internal data
 used for dynamic UI
 managed through hooks like useState
 updated through setter functions
 causes re-rendering
 stored internally by React
 essential for interactive applications

State is the core of React's reactive model.
