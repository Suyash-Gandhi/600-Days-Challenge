
# 38. Advanced Hook Patterns in React

## 1. What Advanced Hook Patterns Are

Advanced hook patterns are techniques for structuring logic with hooks so that:

 Logic becomes reusable
 Components stay clean and readable
 State management becomes scalable
 Performance issues are minimized

Instead of putting everything inside one component, we extract logic into reusable hooks and patterns.

Think of them as architectural patterns for hooks.

---

# 2. Custom Hook Pattern

This is the most common advanced hook pattern.

### Idea

Move reusable logic into a custom hook.

### Example problem

You fetch data in many components.

Bad approach:

Every component repeats:

 state
 loading
 error
 fetch logic

---

### Custom Hook Solution

Create a reusable hook.

Example:

```
function useFetch(url) {
  const [data, setData] = useState(null)
  const [loading, setLoading] = useState(true)

  useEffect(() => {
    fetch(url)
      .then(res => res.json())
      .then(data => {
        setData(data)
        setLoading(false)
      })
  }, [url])

  return { data, loading }
}
```

Usage:

```
const { data, loading } = useFetch("/api/users")
```

### Benefits

 Logic reused
 Components cleaner
 Easier testing
 Easier maintenance

---

# 3. Hook + Context Pattern (Global Logic)

When logic must be shared across the entire app.

Example:

 authentication
 theme
 user data
 global settings

We combine:

```
useContext + custom hook
```

Example:

```
const AuthContext = createContext()

function useAuth() {
  return useContext(AuthContext)
}
```

Usage:

```
const { user } = useAuth()
```

### Why this is powerful

Instead of:

```
useContext(AuthContext)
```

everywhere, we create a clean API.

---

# 4. State Reducer Pattern

This pattern uses useReducer but allows the consumer to control state behavior.

Useful for reusable components or libraries.

Example scenario:

A component manages internal state but allows the parent to modify behavior.

Example:

```
const [state, dispatch] = useReducer(reducer, initialState)
```

But the reducer behavior can be overridden.

Libraries like:

 Downshift
 Reach UI

use this pattern.

---

# 5. Control Props Pattern

This pattern is similar to controlled inputs.

The parent component controls the state instead of the component.

Example:

Instead of this:

```
const [open, setOpen] = useState(false)
```

The component receives:

```
isOpen
onToggle
```

Example:

```
<Modal isOpen={open} onToggle={setOpen} />
```

### Why this matters

Allows components to work in:

 controlled mode
 uncontrolled mode

---

# 6. Compound Component Pattern

This pattern allows multiple components to share state implicitly.

Example:

```
<Tabs>
   <Tabs.List />
   <Tabs.Tab />
   <Tabs.Panel />
</Tabs>
```

These components communicate via context + hooks.

Benefits:

 clean API
 flexible composition
 scalable design systems

Used in:

 Radix UI
 Headless UI
 Reach UI

---

# 7. Render Hook Pattern

Sometimes hooks return functions or render helpers.

Example:

```
const { renderItem } = useList(items)
```

Then:

```
renderItem(item)
```

This pattern allows hooks to control rendering logic.

---

# 8. Hook Factories

A hook factory creates customized hooks.

Example:

```
function createUseFetch(baseUrl) {
  return function useFetch(endpoint) {
    ...
  }
}
```

Usage:

```
const useApi = createUseFetch("/api")
```

This pattern is useful when:

 you need reusable configurations
 you build SDKs
 you build libraries

---

# 9. useEvent Pattern (Stable Event Handlers)

Sometimes callbacks cause unnecessary re-renders.

A pattern is used to create stable event handlers.

Example:

```
const handlerRef = useRef()

handlerRef.current = callback
```

This ensures the function reference doesn't change every render.

---

# 10. Separation of Logic and UI Pattern

Large apps separate:

 logic hooks
 UI components

Example:

Logic hook:

```
useChat()
```

UI component:

```
ChatUI
```

Example usage:

```
const chat = useChat()

return <ChatUI {...chat} />
```

Benefits:

 easier testing
 reusable UI
 reusable logic

---

# 11. Data + Status Hook Pattern

Many hooks return multiple states.

Example:

```
const { data, error, loading } = useQuery()
```

This pattern standardizes:

 loading state
 success state
 error state

Libraries using this:

 React Query
 SWR

---

# 12. When Advanced Hook Patterns Are Needed

They become important when:

Your app has:

 large component trees
 repeated logic
 shared state
 complex UI components
 performance concerns

Without these patterns, apps become:

 hard to maintain
 hard to debug
 messy

---

# 13. Real Industry Example

A chat application may use:

Custom hooks:

```
useMessages()
useUser()
useSocket()
useNotifications()
```

Compound components:

```
<Chat>
   <Chat.Messages />
   <Chat.Input />
</Chat>
```

Context pattern:

```
ChatProvider
```

This structure keeps the project scalable.

---

# Key Takeaway

Advanced hook patterns help React developers:

 build reusable logic
 write clean architecture
 improve performance
 manage complex state

They are essential when building large React applications.
