Debouncing is an optimization technique used in React (and JavaScript generally) to limit how often a function executes when an event fires repeatedly in a short time.

It is extremely common in search inputs, API requests, window resize events, and scroll handlers.

---

# 41. Debouncing in React

## 1. The Problem Debouncing Solves

Many UI events fire very frequently.

Example: typing in a search box.

If a user types:

```
h → he → hel → hell → hello
```

and your React app calls an API on every keystroke, it triggers 5 API calls.

Problems:

 Too many network requests
 Performance issues
 Server overload
 UI lag

Debouncing solves this.

---

## 2. What Debouncing Means

Debouncing delays a function execution until the user stops triggering the event for a specified time.

Example:

```
User types → wait 500ms → run function
```

If the user types again before 500ms finishes, the timer resets.

So the function runs only once after the user pauses typing.

---

### Without Debounce

User types quickly:

```
h      → API call
he     → API call
hel    → API call
hell   → API call
hello  → API call
```

Total calls: 5

---

### With Debounce (500ms)

User types quickly:

```
h
he
hel
hell
hello
(wait 500ms)
→ API call
```

Total calls: 1

---

# 3. Basic Debounce Concept (JavaScript)

Debouncing uses setTimeout + clearTimeout.

Idea:

1. Start a timer when an event occurs
2. If another event occurs before timer finishes → reset timer
3. Only run the function when timer completes

Example:

```javascript
let timer;

const debounce = (func, delay) => {
  clearTimeout(timer);

  timer = setTimeout(() => {
    func();
  }, delay);
};
```

# 4. Debouncing in React (Search Input Example)

Imagine a search bar that fetches results from an API.

### Without Debouncing

```javascript
const Search = () => {
  const [query, setQuery] = useState("");

  const handleChange = (e) => {
    setQuery(e.target.value);
    fetchResults(e.target.value);
  };

  return (
    <input
      type="text"
      value={query}
      onChange={handleChange}
    />
  );
};
```

Problem:

Every keystroke calls:

```
fetchResults()
```

---

# 5. Debouncing with useEffect

A common React approach is debouncing state changes using useEffect.

```javascript
const Search = () => {
  const [query, setQuery] = useState("");
  const [debouncedQuery, setDebouncedQuery] = useState("");

  useEffect(() => {
    const timer = setTimeout(() => {
      setDebouncedQuery(query);
    }, 500);

    return () => clearTimeout(timer);
  }, [query]);

  useEffect(() => {
    if (debouncedQuery) {
      fetchResults(debouncedQuery);
    }
  }, [debouncedQuery]);

  return (
    <input
      value={query}
      onChange={(e) => setQuery(e.target.value)}
    />
  );
};
```

### Flow

1. User types → `query` updates
2. `useEffect` starts 500ms timer
3. If user types again → previous timer clears
4. After user stops typing → `debouncedQuery` updates
5. API call runs once

# 6. Debouncing Using Lodash

Many projects use lodash debounce.

Install:

```
npm install lodash
```

Example:

```javascript
import debounce from "lodash/debounce";

const Search = () => {
  const fetchResults = debounce((value) => {
    console.log("API call:", value);
  }, 500);

  const handleChange = (e) => {
    fetchResults(e.target.value);
  };

  return <input onChange={handleChange} />;
};
```

Important note:

In React this should usually be wrapped with useCallback or useMemo to prevent recreation every render.

# 7. Custom Debounce Hook (Best Practice)

Many React apps create a reusable custom hook.

```javascript
const useDebounce = (value, delay) => {
  const [debouncedValue, setDebouncedValue] = useState(value);

  useEffect(() => {
    const timer = setTimeout(() => {
      setDebouncedValue(value);
    }, delay);

    return () => clearTimeout(timer);
  }, [value, delay]);

  return debouncedValue;
};
```

Usage:

```javascript
const Search = () => {
  const [query, setQuery] = useState("");

  const debouncedQuery = useDebounce(query, 500);

  useEffect(() => {
    fetchResults(debouncedQuery);
  }, [debouncedQuery]);

  return (
    <input
      value={query}
      onChange={(e) => setQuery(e.target.value)}
    />
  );
};
```

Advantages:

 Clean
 Reusable
 Easy to maintain

# 8. Debouncing vs Throttling

These are often confused.

| Feature   | Debouncing        | Throttling          |
| --------- | ----------------- | ------------------- |
| Execution | After event stops | At fixed intervals  |
| Best for  | Search inputs     | Scroll/resize       |
| Calls     | 1 after pause     | Many but controlled |

---

### Example

Typing fast:

Debounce (500ms)

```
h
he
hel
hell
hello
(wait)
→ 1 API call
```

Throttle (500ms)

```
h → API
hel → API
hello → API
```

# 10. Common Mistakes

### 1️⃣ Creating debounce function on every render

Bad:

```javascript
const debounced = debounce(fetch, 500);
```

Every render creates new debounce.

Fix with:

```
useMemo / useCallback
```

---

### 2️⃣ Forgetting cleanup

Timers should always be cleared.

```
return () => clearTimeout(timer);
```

---

### 3️⃣ Debouncing the wrong thing

Debounce user actions, not state updates incorrectly.
