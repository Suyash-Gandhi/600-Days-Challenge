Lazy loading in React Router is an optimization technique where routes (pages) are loaded only when the user visits them, instead of loading everything upfront.

This is very important for performance in real-world apps.

---

# 🔹 1. What is Lazy Loaded Routing?

Normally in React:

 All components (pages) are bundled together
 Even unused pages are downloaded on first load

👉 Problem:

 Bigger bundle size
 Slower initial loading

👉 Solution: Lazy loading

 Load component only when route is visited

---

# 🔹 2. Basic Idea

Instead of this:

```js
import Home from "./pages/Home";
import About from "./pages/About";
```

We do:

```js
const Home = React.lazy(() => import("./pages/Home"));
const About = React.lazy(() => import("./pages/About"));
```

---

# 🔹 3. Complete Example (React Router)

### Step 1: Import required things

```js
import { BrowserRouter, Routes, Route } from "react-router-dom";
import React, { Suspense } from "react";
```

---

### Step 2: Lazy load components

```js
const Home = React.lazy(() => import("./pages/Home"));
const About = React.lazy(() => import("./pages/About"));
```

---

### Step 3: Use Suspense

```js
function App() {
  return (
    <BrowserRouter>
      <Suspense fallback={<h2>Loading...</h2>}>
        <Routes>
          <Route path="/" element={<Home />} />
          <Route path="/about" element={<About />} />
        </Routes>
      </Suspense>
    </BrowserRouter>
  );
}
```

---

# 🔹 4. Why Suspense is Required?

When a component is being lazy loaded:

 It takes time to download
 React needs something to show meanwhile

👉 `Suspense` handles that

```js
<Suspense fallback={<Loader />}>
```

👉 Shows loader until component loads

---

# 🔹 5. How It Works Internally

1. User opens app → only main bundle loads
2. User clicks `/about`
3. React triggers dynamic import:

   ```js
   import("./pages/About")
   ```
4. That file is fetched separately
5. Then component renders

👉 This is called code splitting

---

# 🔹 6. Real-World Example

Imagine:

 Dashboard
 Profile
 Settings
 Admin panel

Without lazy loading:

 All pages load even if user only sees dashboard

With lazy loading:

 Only dashboard loads first
 Others load when needed

---

# 🔹 7. Best Practices

### ✅ 1. Lazy load pages, not small components

 Good: routes/pages
 Bad: buttons, small UI components

---

### ✅ 2. Add proper fallback UI

```js
<Suspense fallback={<Spinner />}>
```

Not:

```js
fallback="Loading..."
```

👉 Use proper loader for better UX

---

### ✅ 3. Group related routes

Instead of too many small chunks, group logically:

 Auth pages
 Admin pages

---

### ✅ 4. Combine with Protected Routes

Lazy loading + protected routes = scalable apps

---

# 🔹 8. Lazy Loading + Protected Route Example

```js
const Dashboard = React.lazy(() => import("./pages/Dashboard"));

<Route
  path="/dashboard"
  element={
    <ProtectedRoute>
      <Dashboard />
    </ProtectedRoute>
  }
/>
```

---

# 🔹 9. Advantages

 Faster initial load
 Better performance
 Reduced bundle size
 Scalable architecture

---

# 🔹 10. Disadvantages

 Slight delay when opening new page
 Requires loading state handling
