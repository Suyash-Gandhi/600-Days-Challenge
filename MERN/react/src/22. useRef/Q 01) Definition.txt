
# 1. The Core Idea of `useRef`

Think of `useRef` as a box that React keeps safe between renders.

Inside that box there is one property:

```
ref.current
```

You can store anything inside it.

Example:

```javascript
const myRef = useRef(0)
```

React creates a box like this:

```
myRef = {
  current: 0
}
```

You can change the value:

```javascript
myRef.current = 5
```

Important:

Changing it does NOT re-render the component.

---

# 2. Why React Created `useRef`

React normally re-renders when state changes.

Example with state:

```javascript
const [count, setCount] = useState(0)
```

If you do:

```
setCount(1)
```

React re-renders the component.

But sometimes we want to store something without re-rendering.

Examples:

 store a timer
 store previous value
 store a DOM element
 store a flag

That is why useRef exists.

---

# 3. Most Common Use: Accessing DOM

Example: focus an input automatically

```javascript
import { useRef, useEffect } from "react"

function App() {

  const inputRef = useRef(null)

  useEffect(() => {
    inputRef.current.focus()
  }, [])

  return <input ref={inputRef} />
}
```

What happens step-by-step:

### Step 1

React creates the ref

```
inputRef = { current: null }
```

---

### Step 2

React attaches the input element

```
inputRef.current = <input DOM element>
```

---

### Step 3

Now we can access the DOM

```
inputRef.current.focus()
```

Result:

The input automatically gets focus.

---

# 4. Another Use: Store Values Without Re-render

Example:

```javascript
const counter = useRef(0)

function increase() {
  counter.current++
  console.log(counter.current)
}
```

Here:

```
counter.current = 1
counter.current = 2
counter.current = 3
```

But the component does not re-render.

---

# 5. Very Important Difference

### `useState`

Used for UI data.

Example:

```javascript
const [count, setCount] = useState(0)
```

Updating:

```
setCount(1)
```

Result:

```
React re-renders
UI updates
```

---

### `useRef`

Used for values that should NOT cause re-render.

Example:

```javascript
const ref = useRef(0)

ref.current = 5
```

Result:

```
No re-render
```

---

# 6. Simple Real Example

Button focuses input.

```javascript
import { useRef } from "react"

function App() {

  const inputRef = useRef(null)

  function focusInput() {
    inputRef.current.focus()
  }

  return (
    <>
      <input ref={inputRef} />
      <button onClick={focusInput}>Focus</button>
    </>
  )
}
```

When button is clicked:

```
focusInput()
↓
inputRef.current
↓
actual input element
↓
.focus()
```

Think of `useRef` as:

```
A container that React keeps between renders.
```

```
useRef = persistent box
```

```
{
  current: value
}
```

And:

```
Changing it does NOT re-render
```

### 2️⃣ Store timer

# 10. One Sentence Interview Answer

`useRef` is a React hook that stores a mutable value that persists across renders without causing a re-render.
