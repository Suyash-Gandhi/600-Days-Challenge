
# 39. Controlled vs Uncontrolled Forms in React

## 1. Controlled Forms

### Definition

A controlled form is a form where React controls the input values using state.

The input field's value is stored in React state, and every change updates that state.

So the flow is:

```
User types → onChange event → update React state → React re-renders input
```

React becomes the single source of truth.

---

### Example

```jsx
import { useState } from "react";

const Form = () => {
  const [name, setName] = useState("");

  return (
    <div>
      <input
        type="text"
        value={name}
        onChange={(e) => setName(e.target.value)}
      />

      <p>Name: {name}</p>
    </div>
  );
};

export default Form;
```

---

### How it works

1. `name` state stores the input value.
2. `value={name}` binds input to React state.
3. `onChange` updates state whenever user types.
4. React re-renders with the new value.

---

### Visualization

```
User Input
     ↓
onChange Event
     ↓
React State Update
     ↓
Component Re-render
     ↓
Updated Input Value
```

---

### Advantages

1. Full control over form data

You can validate input easily.

Example:

```jsx
if(name.length < 3){
   alert("Name too short")
}
```

---

2. Easy validation

```jsx
if (!email.includes("@")) {
  setError("Invalid email");
}
```

---

3. Dynamic UI updates

Example:

Disable button until input valid.

```jsx
<button disabled={!name}>Submit</button>
```

---

4. Predictable state management

All data lives inside React state.

---

### Disadvantages

1. More re-renders

Every keystroke updates state → re-render.

2. More code

Need state + handlers for each field.

---

# 2. Uncontrolled Forms

### Definition

In uncontrolled forms, the DOM itself stores the form values.

React does not control the input state.

Instead, we access the value using refs.

---

### Example

```jsx
import { useRef } from "react";

const Form = () => {
  const inputRef = useRef();

  const handleSubmit = () => {
    alert(inputRef.current.value);
  };

  return (
    <div>
      <input type="text" ref={inputRef} />
      <button onClick={handleSubmit}>Submit</button>
    </div>
  );
};

export default Form;
```

---

### How it works

1. The DOM stores the input value.
2. React only accesses it when needed.
3. We use `ref` to read the value.

---

### Visualization

```
User Input
     ↓
DOM stores value
     ↓
React reads value using ref when needed
```

---

### Advantages

1. Less code

No need for state management.

---

2. Better performance for large forms

Because React does not re-render on every keystroke.

---

3. Useful for quick forms

Example:

 Simple contact form
 File upload input

---

### Disadvantages

1. Harder validation

You must manually read values.

---

2. Harder to manage complex forms

React cannot track input state.

---

3. Less predictable

State is outside React.

---

# Key Differences

| Feature          | Controlled Form      | Uncontrolled Form |
| ---------------- | -------------------- | ----------------- |
| Data storage     | React State          | DOM               |
| Source of truth  | React                | DOM               |
| Value access     | `value` + `onChange` | `ref`             |
| Validation       | Easy                 | Hard              |
| React re-renders | Yes (every change)   | No                |
| Code complexity  | More                 | Less              |

---

# Real World Rule (Very Important)

### Most React apps use Controlled Forms

Because they allow:

 Validation
 Conditional UI
 State management
 Predictable behavior

Examples:

 Login forms
 Signup forms
 Search bars
 Filters
 Multi-step forms

---

### Uncontrolled forms are used when:

 Quick forms
 File uploads
 Integrating with non-React libraries
 Performance sensitive large forms

---

# Example Comparing Both

### Controlled

```jsx
<input
  value={username}
  onChange={(e) => setUsername(e.target.value)}
/>
```

React controls the value.

---

### Uncontrolled

```jsx
<input ref={usernameRef} />
```

DOM controls the value.


---

# Interview Answer (Short Version)

Controlled components are form inputs whose values are managed by React state using `value` and `onChange`.
Uncontrolled components store their values in the DOM, and React accesses them using refs.

Controlled forms provide better validation and control, while uncontrolled forms are simpler and may perform better for large forms.
