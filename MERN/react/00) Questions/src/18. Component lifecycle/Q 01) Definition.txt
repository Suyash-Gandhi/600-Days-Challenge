

# 1️⃣ What is Component Lifecycle?

Every React component goes through 3 major phases:

### 1. Mounting

Component is created and inserted into the DOM.

### 2. Updating

Component re-renders because props or state changed.

### 3. Unmounting

Component is removed from the DOM.

Think of it like:

> Create → Update → Destroy

Now let’s see how this works in both approaches.

---

# 2️⃣ Lifecycle in Class Components

In class components, lifecycle is controlled using special lifecycle methods.

Example structure:

```js
class MyComponent extends React.Component {
  constructor(props) {
    super(props);
    this.state = { count: 0 };
  }

  componentDidMount() {
    console.log("Mounted");
  }

  componentDidUpdate(prevProps, prevState) {
    console.log("Updated");
  }

  componentWillUnmount() {
    console.log("Unmounted");
  }

  render() {
    return <div>{this.state.count}</div>;
  }
}
```

---

## 🔹 Mounting Phase (Class)

1. constructor()
2. render()
3. componentDidMount()

### 🔸 componentDidMount()

 Runs after first render
 Used for:

   API calls
   Subscriptions
   Timers

Example:

```js
componentDidMount() {
  fetch("/api/data")
}
```

---

## 🔹 Updating Phase (Class)

Triggered when:

 setState()
 props change

Order:

1. render()
2. componentDidUpdate()

### 🔸 componentDidUpdate(prevProps, prevState)

Used for:

 Reacting to state/prop changes
 Re-fetching data conditionally

Example:

```js
componentDidUpdate(prevProps) {
  if (prevProps.id !== this.props.id) {
    fetchNewData();
  }
}
```

---

## 🔹 Unmounting Phase (Class)

### 🔸 componentWillUnmount()

Runs right before component is removed.

Used for:

 Clearing intervals
 Removing event listeners
 Cancelling API requests

Example:

```js
componentWillUnmount() {
  clearInterval(this.timer);
}
```

---

# 3️⃣ Lifecycle in Function Components (Hooks Way)

Modern React uses Hooks instead of lifecycle methods.

Main Hook for lifecycle logic:

### 👉 `useEffect()`

This single hook replaces:

 componentDidMount
 componentDidUpdate
 componentWillUnmount

---

## 🔹 Basic Structure

```js
import { useEffect } from "react";

const MyComponent = () => {

  useEffect(() => {
    console.log("Runs after render");
  });

  return <div>Hello</div>;
};
```

---

## 🔹 Mounting (Hooks Way)

To run only once (like componentDidMount):

```js
useEffect(() => {
  console.log("Mounted");
}, []);
```

👉 Empty dependency array = run only once

---

## 🔹 Updating (Hooks Way)

To run when specific value changes:

```js
useEffect(() => {
  console.log("Count changed");
}, [count]);
```

This runs:

 After first render
 Whenever `count` changes

---

## 🔹 Unmounting (Cleanup Function)

```js
useEffect(() => {
  const timer = setInterval(() => {
    console.log("Running");
  }, 1000);

  return () => {
    clearInterval(timer);
    console.log("Cleanup on unmount");
  };
}, []);
```

👉 The returned function acts like `componentWillUnmount`.

---

# 4️⃣ Class vs Hooks — Direct Comparison

| Lifecycle Phase  | Class Component                      | Hooks Way                         |
| ---------------- | ------------------------------------ | --------------------------------- |
| Mount            | componentDidMount                    | useEffect(() => {}, [])           |
| Update           | componentDidUpdate                   | useEffect(() => {}, [dependency]) |
| Unmount          | componentWillUnmount                 | return cleanup function           |
| State            | this.state                           | useState                          |
| Multiple Effects | Need multiple lifecycle logic blocks | Use multiple useEffect            |

---

# 5️⃣ Important Concept: Why Hooks Are Better

### 🔥 1. No duplication

In classes, you often repeat logic:

```js
componentDidMount() { fetchData(); }
componentDidUpdate() { fetchData(); }
```

In hooks:

```js
useEffect(() => {
  fetchData();
}, [id]);
```

Much cleaner.

---

### 🔥 2. Separation of Concerns

You can split effects:

```js
useEffect(() => { document.title = count; }, [count]);

useEffect(() => {
  const id = setInterval(...);
  return () => clearInterval(id);
}, []);
```

Each effect handles one responsibility.

---

### 🔥 3. Functional Components Are Simpler

 Less boilerplate
 No `this`
 Easier testing
 Cleaner logic grouping

That’s why modern React strongly prefers hooks.

---

# 6️⃣ Interview-Level Understanding

If asked:

> How does useEffect replace lifecycle methods?

Answer:

 `useEffect(fn, [])` → componentDidMount
 `useEffect(fn, [dep])` → componentDidUpdate
 `return cleanup` → componentWillUnmount
 Without dependency array → runs after every render

---

# 7️⃣ Best Practices (Very Important)

### ✅ Always clean up:

 Timers
 Event listeners
 WebSocket connections

### ✅ Use correct dependency arrays

Wrong dependencies = bugs or infinite loops.

### ✅ Use multiple useEffect instead of one giant effect.

### ❌ Avoid:

 Putting everything in one useEffect
 Leaving dependency array empty when it shouldn’t be
