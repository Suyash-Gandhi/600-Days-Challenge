#37. useImperativeHandle

`useImperativeHandle` is an advanced React hook used when a parent component needs to directly control a child component using a `ref`.

Normally in React:

 Data flows top в†’ down (props)
 Parent should not directly manipulate child internals

But sometimes you need to expose specific functions from a child to the parent, such as:

 focusing an input
 opening a modal
 resetting a form
 triggering animations

Instead of exposing the entire DOM node, `useImperativeHandle` lets you control exactly what the parent can access.

---

# 1. What `useImperativeHandle` Does

`useImperativeHandle` allows a component to customize the value that is exposed when a parent uses `ref`.

Without it, a `ref` usually points to:

 a DOM element
 or the component instance

With `useImperativeHandle`, you can expose only specific functions or values.

### Basic Idea

Parent:

```
childRef.current.someFunction()
```

Child decides what functions are available through `useImperativeHandle`.

---

# 2. Why React Provides This Hook

React promotes declarative programming.

Example of declarative style:

```
<Modal isOpen={true} />
```

But sometimes you need imperative control.

Example:

```
modalRef.current.open()
modalRef.current.close()
```

`useImperativeHandle` enables safe imperative access while still keeping components encapsulated.

---

# 3. Syntax

```
useImperativeHandle(ref, createHandle, dependencies)
```

### Parameters

1пёЏвѓЈ ref

The ref passed from the parent.

2пёЏвѓЈ createHandle

A function that returns an object containing the exposed methods.

3пёЏвѓЈ dependencies (optional)

Like `useEffect`, determines when the handle updates.

---

# 4. `forwardRef` Is Required

Refs cannot normally be passed to function components.

So you must wrap the component using:

```
React.forwardRef()
```

This allows the parent to pass a `ref` into the child.

---

# 5. Example вЂ” Exposing a Child Function

### Child Component

```
import { forwardRef, useImperativeHandle, useRef } from "react";

const InputComponent = forwardRef((props, ref) => {
  const inputRef = useRef();

  useImperativeHandle(ref, () => ({
    focusInput: () => {
      inputRef.current.focus();
    }
  }));

  return <input ref={inputRef} />;
});

export default InputComponent;
```

---

### Parent Component

```
import { useRef } from "react";
import InputComponent from "./InputComponent";

const App = () => {
  const inputRef = useRef();

  const handleClick = () => {
    inputRef.current.focusInput();
  };

  return (
    <>
      <InputComponent ref={inputRef} />
      <button onClick={handleClick}>Focus Input</button>
    </>
  );
};
```

---

3пёЏвѓЈ Child receives ref via `forwardRef`

# 8. Why Not Just Use DOM Refs?

Without `useImperativeHandle`, the parent would access the DOM directly:

```
inputRef.current.focus()
```

Problem:

Parent now depends on child implementation details.

Example:

If child changes from `input` в†’ `textarea`, the parent breaks.

`useImperativeHandle` hides the internal implementation.

---

# 9. Encapsulation Benefit

Bad approach:

```
parent -> child DOM
```

Good approach:

```
parent -> exposed API
```

Example exposed API:

```
{
 focusInput(),
 clearInput(),
 getValue()
}
```

Parent only knows the API, not the internal structure.

---

# 10. Important Rules

### 1пёЏвѓЈ Must be used with `forwardRef`

---

### 2пёЏвѓЈ Used rarely

React prefers declarative state/props.

Use this hook only when necessary.

---

### 3пёЏвѓЈ Should expose minimal API

Expose only what is required.

Bad:

```

```

```

```
