
# 🚀 44. React Router Basics

## 🔹 1. What is React Router?

React Router is a library that allows you to handle navigation inside a React app without reloading the page.

👉 Normally:

 Clicking a link → full page reload

👉 With React Router:

 Clicking a link → UI updates only (SPA behavior)

---

## 🔹 2. Why React Router is Needed

In React apps:

 Everything runs inside one HTML page
 You don’t have traditional pages like `/about.html`

So React Router helps you:

 Show different components based on URL
 Create navigation like real websites

---

## 🔹 3. Core Concepts (Very Important)

### 1. Router (BrowserRouter)

Wraps your entire app and enables routing.

```jsx
import { BrowserRouter } from "react-router-dom";

<BrowserRouter>
  <App />
</BrowserRouter>
```

👉 Think of it as:

> “Hey React, now this app supports navigation”

---

### 2. Routes & Route

Defines which component should render for which path.

```jsx
import { Routes, Route } from "react-router-dom";

<Routes>
  <Route path="/" element={<Home />} />
  <Route path="/about" element={<About />} />
</Routes>
```

👉 Meaning:

 `/` → Home component
 `/about` → About component

---

### 3. Link (Navigation without reload)

Instead of `<a>` tag, we use `Link`.

```jsx
import { Link } from "react-router-dom";

<Link to="/">Home</Link>
<Link to="/about">About</Link>
```

👉 Why not `<a>`?

 `<a>` reloads page ❌
 `Link` updates UI without reload ✅

---

## 🔹 4. Basic Project Flow (Important Visualization)

Imagine:

```
URL → React Router → Match Route → Render Component
```

Example:

 User goes to `/about`
 React Router checks routes
 Finds `/about`
 Renders `<About />`

---

## 🔹 5. Full Simple Example

```jsx
import { BrowserRouter, Routes, Route, Link } from "react-router-dom";

const Home = () => <h1>Home Page</h1>;
const About = () => <h1>About Page</h1>;

function App() {
  return (
    <BrowserRouter>
      <nav>
        <Link to="/">Home</Link>
        <Link to="/about">About</Link>
      </nav>

      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
      </Routes>
    </BrowserRouter>
  );
}
```

---

## 🔹 6. Important Points (Interview + Real Use)

### ✅ React Router is client-side routing

 No server request for page change

### ✅ URL still changes

 Helps with bookmarking & sharing links

### ✅ Only matching route renders

 Efficient rendering

---

## 🔹 7. Common Mistakes

### ❌ Using `<a>` instead of `<Link>`

→ Causes full reload

---

### ❌ Forgetting `<BrowserRouter>`

→ Routes won’t work

---

### ❌ Using `component` instead of `element` (old version)

```jsx
// ❌ Old
<Route path="/" component={Home} />

// ✅ New
<Route path="/" element={<Home />} />
```
