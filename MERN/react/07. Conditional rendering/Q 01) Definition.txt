## 7. Conditional Rendering Techniques in React

Conditional rendering in React means displaying different UI elements based on certain conditions (state, props, authentication status, API data, etc.).

Used outside JSX.

### Example:


# ðŸ”¹ 2. Ternary Operator (Most Common Method)

Syntax:

```js
condition ? truePart : falsePart
```

### Example:

```jsx

  const isLoggedIn = false;

  return (
    <div>
      {isLoggedIn ? <h1>Dashboard</h1> : <h1>Login Page</h1>}
    </div>
  );
};
```

### Why This is Popular:

 Clean
 Compact

# ðŸ”¹ 3. Logical AND (&&) Operator

Used when you want to render something only if condition is true.

### Example:

```jsx
const App = () => {
  const hasNotification = true;

  return (
    <div>
      <h1>Profile</h1>
      {hasNotification && <p>You have new messages!</p>}
    </div>
  );
};
```

If condition is false â†’ nothing renders.

### Best Use Case:

 Showing alerts
 Showing badges
 Showing error messages
 Optional UI parts

# ðŸ”¹ 4. Logical OR (||) Operator

Used to show fallback UI.

```jsx
const App = () => {
  const username = "";

  return (
    <div>
      <h1>Hello {username || "Guest"}</h1>
    </div>
  );
};
```

If `username` is empty â†’ "Guest" will display.

# ðŸ”¹ 5. Element Variables

Store JSX inside variables and render conditionally.

```jsx
const App = () => {
  const isAdmin = true;
  let message;

  if (isAdmin) {
    message = <h2>Admin Panel</h2>;
  } else {
    message = <h2>User Panel</h2>;
  }

  return <div>{message}</div>;
};
```

### Advantage:

 Clean separation
 Better readability for complex UI

---

# ðŸ”¹ 6. Switch Case (Multiple Conditions)

Used when there are multiple states.

```jsx
const App = () => {
  const status = "loading";

  switch (status) {
    case "loading":
      return <h1>Loading...</h1>;
    case "success":
      return <h1>Data Loaded</h1>;
    case "error":
      return <h1>Something went wrong</h1>;
    default:
      return null;
  }
};
```

### Best for:

 API states
 User roles
 Page modes

# ðŸ”¥ Best Practices (Very Important)

### âœ… 1. Use Ternary for Small Conditions

### âœ… 2. Use && for Optional UI

### âœ… 3. Use if-else or switch for large JSX blocks

### âœ… 4. Avoid deeply nested ternaries

### âœ… 5. Extract complex conditions into functions

Example:

```jsx

  if (loading) return <p>Loading...</p>;
  if (error) return <p>Error occurred</p>;
  return <p>Data loaded</p>;
};

return <div>{renderContent()}</div>;
```

Cleaner and scalable.
