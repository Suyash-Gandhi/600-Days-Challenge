### 10. One-way Data Flow Principle

The one-way data flow principle is a core idea in React (and modern UI frameworks) that defines how data moves inside an application. Understanding this deeply will save you from bugs, confusion, and messy state management later.

## How data actually moves in React

### 1. Parent holds the state

The parent component:

 Creates the data (state)
 Decides how it changes

### 2. Parent passes data to child

The child:

 Receives data via props
 Treats it as read-only

### 3. Child cannot modify parent data directly

If a child needs to change something:

 It asks the parent
 By calling a function passed from the parent

## What React forbids (and why)

React does not allow:

 Child components modifying props
 Two components directly sharing mutable data
 Bidirectional data binding (like old Angular)

Why?
Because those patterns:

 Cause unpredictable UI
 Make bugs extremely hard to track
 Break component isolation

| --------------- | --------------- | ------------- |
| Data direction  | Parent â†’ Child  | Both ways     |
| Debugging       | Easy            | Hard          |
| Predictability  | High            | Low           |
| State ownership | Clear           | Confusing     |

React intentionally chose one-way flow for long-term maintainability.
