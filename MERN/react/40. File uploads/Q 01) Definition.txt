In React applications, file uploads allow users to select files (images, PDFs, documents, etc.) from their device and send them to a server or process them in the browser. This is commonly used in features like profile picture uploads, document submission forms, image galleries, and chat attachments.

Below is a complete explanation of how file uploads work in React, including the concepts, workflow, and best practices.

---

# 40. File Uploads in React

## 1. What File Upload Means

File upload refers to the process where:

1. A user selects a file from their device.
2. The browser reads the file.
3. The file is either:

    Sent to a backend server, or
    Processed inside the frontend.

Example uses:

 Uploading profile pictures
 Uploading resume PDFs
 Uploading images in social media
 Sending attachments in chat apps

---

# 2. Basic File Input in React

File uploads start with the HTML input element:

```html
<input type="file" />
```

Important properties:

| Property    | Purpose                         |
| ----------- | ------------------------------- |
| type="file" | Enables file selection          |
| accept      | Restricts file type             |
| multiple    | Allows selecting multiple files |

Example:

```jsx
<input type="file" accept="image/" />
```

This allows only images.

---

# 3. Handling File Selection in React

When a user selects a file, React receives it through an event object.

The file is accessed through:

```
event.target.files
```

This is a FileList object.

Example structure:

```
FileList
  ├── File 0
  ├── File 1
  └── File 2
```

Each file contains:

| Property     | Description |
| ------------ | ----------- |
| name         | file name   |
| size         | file size   |
| type         | MIME type   |
| lastModified | timestamp   |

Example file object:

```
{
  name: "photo.png",
  size: 24567,
  type: "image/png"
}
```

---

# 4. Storing the File in State

Usually, the selected file is stored in React state so it can later be uploaded.

Example flow:

1. User selects file
2. `onChange` event fires
3. File is stored in state
4. Submit button uploads file

Conceptual example:

```jsx
const [file, setFile] = useState(null)
```

Then inside the handler:

```jsx
const handleChange = (e) => {
  setFile(e.target.files[0])
}
```

Why `files[0]`?

Because `files` is an array-like list.

---

# 5. Uploading Files to a Server

Files cannot be sent as normal JSON.

Instead we use FormData.

FormData allows sending:

 files
 text
 form fields

Example structure:

```
FormData
   ├── file → image.png
   ├── name → "Suyash"
   └── age → 22
```

Typical upload flow:

1. Create FormData
2. Append file
3. Send via API request

Example concept:

```jsx
const formData = new FormData()
formData.append("file", file)
```

Then send using:

 fetch
 axios

# 7. Previewing Files Before Upload

Often users want to preview the image before uploading.

This is done using:

```
URL.createObjectURL(file)
```

It creates a temporary URL.

Example flow:

```
User selects image
      ↓
React generates preview URL
      ↓
Image displayed in UI
```

Common in:

 Profile picture uploads
 Instagram posts
 Product listings

---

# 8. Multiple File Uploads

If the input has `multiple`, users can upload several files.

```
<input type="file" multiple />
```

Then:

```
event.target.files
```

contains multiple files.

Example:

```
FileList
   ├── image1.png
   ├── image2.png
   └── image3.png
```

You usually loop through them before uploading.

Use cases:

 Photo gallery uploads
 Document submission
 Media attachments

```
onUploadProgress
```

Used for:

 showing progress bars

# 10. Client-Side File Validation

Before uploading, files should be validated.

Common checks:

### File Size

Example rule:

```
Max size: 5MB
```

If larger → reject.

---

### File Type

Check MIME type:

```
image/png
image/jpeg
application/pdf
```

Reject unwanted types.

---

### File Count

Example:

```
Max 5 files
```

This prevents abuse.

---

# 11. Security Considerations

File uploads can be dangerous.

Important protections:

### 1. Validate on server

Frontend validation is not enough.

### 2. Restrict file types

Example allowed types:

```
png
jpg
pdf
```

### 3. Limit file size

Large uploads can crash servers.

### 4. Scan files

Some systems scan uploads for malware.
