

# 🔹 1. What Are Nested Routes?

### 👉 Simple Definition:

Nested routes allow you to render a route inside another route (parent → child UI).

Instead of replacing the entire page, only a part of the UI changes.

---

### 👉 Real-life Example:

Think of a dashboard:

```
/dashboard
/dashboard/profile
/dashboard/settings
```

 Sidebar stays same
 Only main content changes

👉 That’s nested routing.

---

# 🔹 2. Why We Use Nested Routes

Without nested routes:

 Entire page re-renders
 Layout gets duplicated
 Hard to maintain

With nested routes:

 Shared layout (Navbar, Sidebar)
 Clean structure
 Scalable apps

---

# 🔹 3. Basic Concept (Visual)

```
Parent Route (Layout)
   ├── Child Route 1
   ├── Child Route 2
   └── Child Route 3
```

👉 Parent renders common UI
👉 Child renders dynamic content

---

# 🔹 4. How Nested Routes Work (Core Idea)

React Router uses a special component:

### 👉 `Outlet`

👉 This is the placeholder where child routes render

---

# 🔹 5. Basic Example (Important)

## Step 1: Parent Layout

```jsx
import { Outlet, Link } from "react-router-dom";

const Dashboard = () => {
  return (
    <div>
      <h1>Dashboard</h1>

      <nav>
        <Link to="profile">Profile</Link>
        <Link to="settings">Settings</Link>
      </nav>

      {/ Child routes render here /}
      <Outlet />
    </div>
  );
};

export default Dashboard;
```

---

## Step 2: Child Components

```jsx
const Profile = () => <h2>Profile Page</h2>;
const Settings = () => <h2>Settings Page</h2>;
```

---

## Step 3: Route Configuration

```jsx
import { Routes, Route } from "react-router-dom";

<Routes>
  <Route path="dashboard" element={<Dashboard />}>
    <Route path="profile" element={<Profile />} />
    <Route path="settings" element={<Settings />} />
  </Route>
</Routes>
```

---

# 🔹 6. How It Actually Works (Flow)

When user visits:

### 👉 `/dashboard/profile`

1. React loads `<Dashboard />`
2. Finds `<Outlet />`
3. Injects `<Profile />` inside it

👉 Final UI:

```
Dashboard
  - Navbar
  - Profile content (inside Outlet)
```

---

# 🔹 7. Important Rule (VERY IMPORTANT ⚠️)

### Child routes are relative paths

```jsx
<Route path="dashboard">
  <Route path="profile" />   // ✅ NOT "/profile"
</Route>
```

👉 React automatically builds:

```
dashboard/profile
```

---

# 🔹 8. Default Child Route (Index Route)

Sometimes you want:

```
/dashboard → show something by default
```

### 👉 Use `index`

```jsx
<Route path="dashboard" element={<Dashboard />}>
  <Route index element={<h2>Welcome Dashboard</h2>} />
  <Route path="profile" element={<Profile />} />
</Route>
```

👉 `/dashboard` → shows default content

---

# 🔹 9. Nested Layouts (Advanced but Important)

You can nest multiple levels:

```jsx
<Route path="dashboard" element={<Dashboard />}>
  <Route path="settings" element={<Settings />}>
    <Route path="account" element={<Account />} />
  </Route>
</Route>
```

👉 URL:

```
/dashboard/settings/account
```

👉 UI:

 Dashboard layout
 Settings layout
 Account content
