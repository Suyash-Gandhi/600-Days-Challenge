# 19️⃣ `useEffect` in React

## 1️⃣ What is `useEffect`?

`useEffect` is a **React Hook** that allows you to perform side effects in function components.

### 🔹 What is a “side effect”?

A side effect is anything that:

* Interacts with the outside world
* Doesn’t just calculate and return JSX
* Happens *after* rendering

Examples:

* Fetching API data
* Setting up event listeners
* Starting timers
* Updating `document.title`
* Subscribing to WebSocket
* Manually manipulating DOM

React renders UI → then runs effects.

---

## 2️⃣ Basic Syntax of `useEffect`

```javascript
import { useEffect } from "react";

useEffect(() => {
  // side effect logic

  return () => {
    // cleanup logic (optional)
  };
}, [dependencies]);
```

### Structure:

```javascript
useEffect(() => {
  // 1. Effect logic runs

  return () => {
    // 2. Cleanup logic runs before next effect or unmount
  };
}, [dependency1, dependency2]);
```

---

# 3️⃣ When Does `useEffect` Run?

It depends on the **dependency array**.

---

## Case 1: No dependency array

```javascript
useEffect(() => {
  console.log("Runs after every render");
});
```

✔ Runs after every render
⚠ Dangerous if it updates state → infinite loop

---

## Case 2: Empty dependency array `[]`

```javascript
useEffect(() => {
  console.log("Runs only once");
}, []);
```

✔ Runs only once (after first render)
✔ Equivalent to `componentDidMount` in class components

Used for:

* API calls
* Setting up subscriptions
* Initial logic

---

## Case 3: With dependencies `[value]`

```javascript
useEffect(() => {
  console.log("Runs when value changes");
}, [value]);
```

✔ Runs:

* On first render
* Whenever `value` changes

---

# 4️⃣ Real Example – API Fetching

```javascript
import { useState, useEffect } from "react";

const User = () => {
  const [user, setUser] = useState(null);

  useEffect(() => {
    const fetchUser = async () => {
      const response = await fetch("https://jsonplaceholder.typicode.com/users/1");
      const data = await response.json();
      setUser(data);
    };

    fetchUser();
  }, []);

  return (
    <div>
      {user ? <h2>{user.name}</h2> : <p>Loading...</p>}
    </div>
  );
};

export default User;
```

✔ Runs once
✔ Fetches data
✔ Updates state

# 5️⃣ Cleanup Function

Cleanup runs:

* Before next effect execution
* When component unmounts

---

## Example 1: Timer Cleanup

```javascript
import { useEffect, useState } from "react";

const Timer = () => {
  const [count, setCount] = useState(0);

  useEffect(() => {
    const interval = setInterval(() => {
      setCount(prev => prev + 1);
    }, 1000);

    return () => {
      clearInterval(interval);
    };
  }, []);

  return <h2>{count}</h2>;
};
```

Without cleanup → memory leak
With cleanup → interval cleared on unmount

---

## Example 2: Event Listener Cleanup

```javascript
useEffect(() => {
  const handleResize = () => {
    console.log(window.innerWidth);
  };

  window.addEventListener("resize", handleResize);

  return () => {
    window.removeEventListener("resize", handleResize);
  };
}, []);
```

Always remove listeners in cleanup.

# 6️⃣ Dependency Array – Deep Understanding

React checks dependencies using **Object.is comparison**.

It compares:

* Previous dependency value
* New dependency value

If changed → effect runs again.

---

### Example

```javascript
useEffect(() => {
  console.log("Count changed");
}, [count]);
```

If `count` changes → effect runs.

If not → skipped.

# 7️⃣ Common Pitfalls (VERY IMPORTANT)

---

## ⚠ 1. Infinite Re-render Loop

```javascript
useEffect(() => {
  setCount(count + 1);
}, [count]);
```

What happens?

1. count changes
2. effect runs
3. setCount updates count
4. count changes again
5. effect runs again

Infinite loop.

---

## ⚠ 2. Missing Dependencies

```javascript
useEffect(() => {
  console.log(userId);
}, []);
```

If `userId` changes → effect does NOT run.

This causes stale values.

Correct:

```javascript
useEffect(() => {
  console.log(userId);
}, [userId]);
```

---

## ⚠ 3. Using Objects or Functions as Dependencies

```javascript
const obj = { name: "John" };

useEffect(() => {
  console.log(obj);
}, [obj]);
```

Problem:

Object is recreated on every render → effect runs every time.

Solution:

Use `useMemo` or define outside component.

---

## ⚠ 4. Stale Closures

```javascript
useEffect(() => {
  const interval = setInterval(() => {
    console.log(count);
  }, 1000);
}, []);
```

If count updates → effect does NOT re-run
Interval keeps logging old value.

Fix:

```javascript
useEffect(() => {
  const interval = setInterval(() => {
    console.log(count);
  }, 1000);

  return () => clearInterval(interval);
}, [count]);
```
