# Middleware Execution Order in Express.js

Middleware execution order is one of the most important concepts in Express.js because Express processes requests from top to bottom in the exact order middleware and routes are written.

Think of middleware like checkpoints in a pipeline.

When a request comes in:

1. Express starts from the top of the file
2. Executes middleware one by one
3. Moves to the next middleware using `next()`
4. Stops when:

    a response is sent, OR
    an error occurs, OR
    no matching route is found

---

# Why Execution Order Matters

The order decides:

 Which middleware runs first
 Whether authentication happens before routes
 Whether request body is available
 Whether logging occurs
 Whether static files work properly
 Whether errors are caught

A wrong order can completely break the app.

---

# Basic Flow

```js
app.use(middleware1)
app.use(middleware2)

app.get("/", handler)
```

Execution flow:

```txt
Request →
middleware1 →
middleware2 →
route handler →
Response
```

---

# Example 1 — Simple Middleware Order

```js
const express = require("express")
const app = express()

app.use((req, res, next) => {
    console.log("Middleware 1")
    next()
})

app.use((req, res, next) => {
    console.log("Middleware 2")
    next()
})

app.get("/", (req, res) => {
    console.log("Route Handler")
    res.send("Hello")
})

app.listen(3000)
```

If user visits `/`

Console output:

```txt
Middleware 1
Middleware 2
Route Handler
```

This proves:

 Express executes in sequence
 Top → Bottom

---

# Visual Understanding

```txt
Incoming Request
       ↓
Middleware 1
       ↓
Middleware 2
       ↓
Route Handler
       ↓
Response Sent
```

# What Happens if `next()` is Not Called?

Middleware chain stops.

Example:

```js
app.use((req, res, next) => {
    console.log("Middleware 1")
})

app.use((req, res, next) => {
    console.log("Middleware 2")
    next()
})
```

Result:

```txt
Middleware 1
```

Request hangs forever because:

 Middleware 1 neither:

   sends response
   nor calls `next()`

So Express cannot continue.

---

# Middleware That Ends Request

Some middleware intentionally stop the chain.

Example:

```js
app.use((req, res, next) => {
    res.send("Request ended")
})

app.get("/", (req, res) => {
    res.send("Hello")
})
```

Route handler never runs.

Because response already sent.

---

# Real-World Middleware Execution Order

A professional Express app usually follows this order:

```js
const express = require("express")
const app = express()

// 1. Logging middleware
app.use(logger)

// 2. Body parser
app.use(express.json())

// 3. Authentication middleware
app.use(authMiddleware)

// 4. Routes
app.use("/api/users", userRoutes)

// 5. Error middleware
app.use(errorHandler)
```

---

# Why This Order?

## 1. Logger First

```js
app.use(logger)
```

So every request gets logged.

---

## 2. Body Parser Before Routes

```js
app.use(express.json())
```

Without this:

```js
req.body
```

will be `undefined`.

Routes need parsed body first.

---

## 3. Authentication Before Protected Routes

```js
app.use(authMiddleware)
```

Checks user before accessing routes.

---

## 4. Routes After Middleware

Routes depend on previous middleware.

---

## 5. Error Middleware Last

Error handlers must be at the end.

Because errors travel downward.

---

# Execution Order with Route-Specific Middleware

Example:

```js
app.use((req, res, next) => {
    console.log("Global Middleware")
    next()
})

app.get(
    "/dashboard",

    (req, res, next) => {
        console.log("Route Middleware")
        next()
    },

    (req, res) => {
        console.log("Route Handler")
        res.send("Dashboard")
    }
)
```

Output:

```txt
Global Middleware
Route Middleware
Route Handler
```

---

# Multiple Middleware in One Route

```js
app.get(
    "/",

    middleware1,
    middleware2,

    (req, res) => {
        res.send("Done")
    }
)
```

Execution:

```txt
middleware1 →
middleware2 →
handler
```

---

# Execution Order with Routers

Example:

```js
app.use("/users", userRouter)
```

Inside router:

```js
router.use((req, res, next) => {
    console.log("Router Middleware")
    next()
})

router.get("/", (req, res) => {
    res.send("Users")
})
```

Request to `/users`

Execution:

```txt
App Middleware →
Router Middleware →
Route Handler
```

---

# Error Middleware Execution Order

Error middleware only runs when:

```js
next(error)
```

is called.

Example:

```js
app.use((req, res, next) => {
    next(new Error("Something broke"))
})

app.use((err, req, res, next) => {
    console.log(err.message)
    res.status(500).send("Error")
})
```

Execution:

```txt
Normal Middleware →
Error Middleware
```

---

# Important Rule About Error Middleware

Error middleware must have 4 parameters:

```js
(err, req, res, next)
```

Otherwise Express treats it as normal middleware.

---

# Static Middleware Order

Example:

```js
app.use(express.static("public"))

app.get("/", (req, res) => {
    res.send("Home")
})
```

If file exists in `public`,
Express serves file first.

Route may never execute.
