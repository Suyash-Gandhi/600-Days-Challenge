# `app.listen()` and Ports in Express.js

When you create an Express server, it will not start automatically.
You must tell it:

 which port to run on
 start listening for incoming requests

That is done using:

```js
app.listen()
```

---

# What is a Port?

A port is like a door or channel through which your application communicates.

Your computer can run multiple applications at the same time:

 Chrome
 VS Code
 MongoDB
 Node.js server

So the operating system uses ports to identify which app should receive the request.

Example:

| Application    | Port  |
| -------------- | ----- |
| React App      | 3000  |
| Express Server | 5000  |
| MongoDB        | 27017 |

---

# Basic Syntax

```js
app.listen(port, callback)
```

---

# Example

```js
import express from "express"

const app = express()

app.listen(5000, () => {
    console.log("Server is running on port 5000")
})
```

---

# Breakdown

## `5000`

This is the port number.

It means:

> "Run this server on port 5000"

Your server URL becomes:

```txt
http://localhost:5000
```

---

## Callback Function

```js
() => {
   console.log("Server is running")
}
```

This runs after the server starts successfully.

Used for:

 confirmation messages
 database connection logs
 debugging

---

# Full Example

```js
import express from "express"

const app = express()

app.get("/", (req, res) => {
    res.send("Home Route")
})

app.listen(5000, () => {
    console.log("Server running on port 5000")
})
```

---

# What Happens Internally?

When you run:

```bash
node index.js
```

Node.js:

1. Creates the Express server
2. Opens port `5000`
3. Starts waiting for requests
4. Responds when someone visits the route

---

# Testing in Browser

If server runs on:

```txt
http://localhost:5000
```

And route is:

```js
app.get("/")
```

Then opening:

```txt
http://localhost:5000
```

will show:

```txt
Home Route
```

# Better Practice Using Variables

Instead of hardcoding:

```js
app.listen(5000)
```

use:

```js
const PORT = 5000

app.listen(PORT, () => {
    console.log(`Server running on port ${PORT}`)
})
```

This makes code cleaner and easier to change later.

---

# Using Environment Variables

In real projects:

```js
const PORT = process.env.PORT || 5000
```

Meaning:

 use environment port if available
 otherwise use `5000`

Example:

```js
app.listen(process.env.PORT || 5000)
```

This is important when deploying apps.

---

# Common Beginner Errors

## 1. Port Already in Use

Error:

```txt
EADDRINUSE
```

Meaning another app is already using that port.

Solution:

 stop the other server
 or change the port

Example:

```js
app.listen(8000)
```

---

## 2. Forgetting to Start Server

If you only write routes:

```js
app.get("/")
```

but no:

```js
app.listen()
```

the server will never run.

---

## 3. Wrong URL

If port is `5000`:

Correct:

```txt
http://localhost:5000
```

Wrong:

```txt
http://localhost:3000
```

```js
import express from "express"

```

# Interview Explanation

If interviewer asks:

> What is `app.listen()`?

You can say:

> `app.listen()` starts the Express server and makes it listen for incoming client requests on a specific port.
