## ЁЯФЯ `*args` and `**kwargs` in Python

When working with functions in Python, sometimes you donтАЩt know in advance how many arguments will be passed. ThatтАЩs where `**args` and `**kw**args` come in тАФ they allow your function to handle a variable number of inputs in a clean and flexible way.

---

# ЁЯЯв 1. What is `*args`?

### ЁЯСЙ Definition

`*args` allows a function to accept any number of positional arguments.

 тАЬ*argsтАЭ stands for arguments
 The `` packs all extra positional arguments into a tuple

---

### ЁЯСЙ Example

```python
def add_numbers(*args):
    print(*args)

add_numbers(1, 2, 3, 4)
```

ЁЯСЙ Output:

```
(1, 2, 3, 4)
```

тЬФ All values are stored in a tuple

---

### ЁЯСЙ Real Use Case

```python
def sum_all(*args):
    total = 0
    for num in **args:
        total += num
    return total

print(sum_all(1, 2, 3))        # 6
print(sum_all(5, 10, 15, 20))  # 50
```

тЬФ You can pass any number of numbers

---

### ЁЯСЙ Key Points

 Accepts positional arguments
 Stored as a tuple
 Can be named anything (`numbers`, `values`), but `**args` is standard

---

# ЁЯЯв 2. What is `**kwargs`?

### ЁЯСЙ Definition

`**kwargs` allows a function to accept any number of keyword arguments (key-value pairs).

 тАЬ**kwargsтАЭ = keyword arguments
 The `` packs them into a dictionary

---

### ЁЯСЙ Example

```python
def print_details(**kwargs):
    print(**kwargs)

print_details(name="Suyash", age=21)
```

ЁЯСЙ Output:

```
{'name': 'Suyash', 'age': 21}
```

тЬФ Stored as a dictionary

---

### ЁЯСЙ Real Use Case

```python
def user_info(**kwargs):
    for key, value in **kwargs.items():
        print(f"{key}: {value}")

user_info(name="Suyash", age=21, city="Delhi")
```

тЬФ Flexible user data handling

---

### ЁЯСЙ Key Points

 Accepts keyword arguments
 Stored as a dictionary
 Can be renamed (`data`, `info`), but `**kwargs` is standard

---

# ЁЯЯв 3. Difference Between `*args` and `**kwargs`

| Feature        | `*args`                  | `**kwargs`               |
| -------------- | ------------------------ | ------------------------ |
| Type           | Positional arguments     | Keyword arguments        |
| Data structure | Tuple                    | Dictionary               |
| Example input  | `(1, 2, 3)`              | `{name="Suyash"}`        |
| Usage          | Unknown number of values | Unknown named parameters |

---

# ЁЯЯв 4. Using Both Together

You can use both in the same function:

```python
def demo(a, *args, **kwargs):
    print("a:", a)
    print("args:", *args)
    print("kwargs:", **kwargs)

demo(1, 2, 3, name="Suyash", age=21)
```

ЁЯСЙ Output:

```
a: 1
args: (2, 3)
kwargs: {'name': 'Suyash', 'age': 21}
```

---

### ЁЯСЙ Order Rule (IMPORTANT тЪая╕П)

```python
def func(a, *args, **kwargs):
    pass
```

тЬФ Correct order:

1. Normal parameters
2. `*args`
3. `**kwargs`

---

# ЁЯЯв 5. Unpacking with `*` and `**`

These symbols are also used to unpack values.

---

### ЁЯСЙ Example with ``

```python
numbers = [1, 2, 3]
print(numbers)
```

ЁЯСЙ Output:

```
1 2 3
```

---

### ЁЯСЙ Example with ``

```python
data = {"name": "Suyash", "age": 21}

def show(name, age):
    print(name, age)

show(data)
```

тЬФ Dictionary unpacked into arguments

---

# ЁЯЯв 6. When to Use (Practical Understanding)

Use `**args` when:

 You donтАЩt know how many positional inputs
 Example: sum, logging, flexible input

Use `**kwargs` when:

 You want optional named parameters
 Example: config settings, user data

---

# ЁЯЯв 7. Real-World Example (Very Important)

```python
def create_profile(name, *skills, **details):
    print("Name:", name)
    print("Skills:", skills)
    print("Details:", details)

create_profile(
    "Suyash",
    "Python", "React",
    age=21,
    city="Delhi"
)
```

ЁЯСЙ Output:

```
Name: Suyash
Skills: ('Python', 'React')
Details: {'age': 21, 'city': 'Delhi'}
```

тЬФ This pattern is heavily used in:

 Frameworks (like Django, Flask)
 APIs
 Config-driven systems
