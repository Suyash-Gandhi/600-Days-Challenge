
#3 What are Truthy and Falsy values?

In Python, every value is automatically interpreted as either:

 Truthy → behaves like `True`
 Falsy → behaves like `False`

👉 You don’t always need to write `== True` or `== False`
Python implicitly converts values to boolean when needed.

---

# 🟢 Falsy Values (Important List)

These values are considered False in Python:

### 1. Boolean False

```python
False
```

### 2. Zero values

```python
0
0.0
0j   # complex zero
```

### 3. Empty sequences / collections

```python
""      # empty string
[]      # empty list
()      # empty tuple
{}      # empty dictionary
set()   # empty set
```

### 4. None (absence of value)

```python
None
```

👉 These are the only falsy values you need to remember

---

# 🟢 Truthy Values

Everything else is truthy.

Examples:

```python
1
-10
3.14
"hello"
[1, 2]
{"a": 1}
True
```

👉 Even weird things like:

```python
"0"   # string → truthy
[0]   # list with zero → truthy
```

---

# 🟢 How Python Uses This (Very Important)

### Example 1: If condition

```python
name = ""

if name:
    print("Has value")
else:
    print("Empty")
```

👉 Output:

```
Empty
```

Because `""` is falsy.

---

### Example 2: Checking list

```python
items = [1, 2, 3]

if items:
    print("List is not empty")
```

👉 No need to write:

```python
if len(items) > 0   # unnecessary
```

---

### Example 3: Using None

```python
data = None

if not data:
    print("No data found")
```

---

# 🟢 Real-World Use Cases (Important for Projects)

### 1. Form validation

```python
if not username:
    print("Username required")
```

### 2. API response check

```python
if response:
    process(response)
```

### 3. Default values

```python
name = user_input or "Guest"
```

👉 If `user_input` is falsy → `"Guest"` is used

---

# 🟢 Key Difference (Beginner Confusion)

### ❌ Wrong thinking:

"Falsy means False only"

### ✅ Correct thinking:

Falsy means:

> “Python treats this value as False in a boolean context”

---

# 🟢 Best Practices

 ✅ Use truthy/falsy checks instead of long conditions
 ❌ Avoid unnecessary comparisons:

```python
if x == True     # bad
if x             # good
```

 ✅ Use `not` for clean checks:

```python
if not items:
```

---

# 🟢 Quick Summary

 Python automatically converts values to `True` or `False`
 Falsy values:

   `False`, `0`, `None`, empty collections
 Truthy values:

   Everything else
 Used heavily in:

   `if` statements
   loops
   validations
   shortcuts
