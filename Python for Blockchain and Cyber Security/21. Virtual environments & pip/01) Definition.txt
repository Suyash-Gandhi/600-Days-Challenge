### 21. Virtual Environments & `pip` (Python)


# 🔹 1. What is a Virtual Environment?

A virtual environment is an isolated Python environment where you can install packages without affecting the global system Python.

### 👉 Why it exists

 Different projects need different versions of libraries
 Installing everything globally creates conflicts

### Example:

 Project A needs `Django 3`
 Project B needs `Django 5`

Without virtual environments → conflict
With virtual environments → both work independently

---

# 🔹 2. Why You MUST Use Virtual Environments

### ✔ Benefits

 Isolation → no dependency conflicts
 Clean projects → each project has its own setup
 Reproducibility → others can recreate your environment
 Safe updates → you don’t break other projects

---

# 🔹 3. Creating a Virtual Environment

Python has a built-in module: `venv`

### 👉 Command:

```bash
python -m venv myenv
```

This creates a folder:

```
myenv/
  ├── Scripts/ (Windows)
  ├── bin/ (Mac/Linux)
  └── lib/
```

---

# 🔹 4. Activating the Virtual Environment

### ▶ Windows:

```bash
myenv\Scripts\activate
```

### ▶ Mac/Linux:

```bash
source myenv/bin/activate
```

### ✔ After activation:

Your terminal shows:

```
(myenv) C:\project>
```

---

# 🔹 5. Deactivating

```bash
deactivate
```

---

# 🔹 6. What is `pip`?

`pip` = Python package manager

Used to:

 Install packages
 Update packages
 Remove packages

---

# 🔹 7. Basic `pip` Commands

### ▶ Install a package

```bash
pip install requests
```

### ▶ Install specific version

```bash
pip install django==4.2
```

### ▶ Upgrade package

```bash
pip install --upgrade requests
```

### ▶ Uninstall

```bash
pip uninstall requests
```

### ▶ List installed packages

```bash
pip list
```

---

# 🔹 8. Requirements File (VERY IMPORTANT)

This is how real projects manage dependencies.

### ▶ Create:

```bash
pip freeze > requirements.txt
```

Example:

```
django==4.2
requests==2.31.0
```

### ▶ Install from file:

```bash
pip install -r requirements.txt
```

👉 This ensures same setup everywhere

---

# 🔹 9. Project Workflow (Real-world flow)

### Step-by-step:

```bash
# 1. Create env
python -m venv venv

# 2. Activate
venv\Scripts\activate

# 3. Install packages
pip install flask

# 4. Save dependencies
pip freeze > requirements.txt
```

---

# 🔹 10. Best Practices

### ✔ Naming

 Use `venv/` or `.venv/`

### ✔ Always ignore in Git

Add to `.gitignore`:

```
venv/
```

### ✔ Always use requirements.txt

Never rely on memory

### ✔ One env per project

Never share environments

---

# 🔹 11. Common Mistakes

### ❌ Installing globally

→ Causes conflicts

### ❌ Forgetting to activate env

→ Packages install in wrong place

### ❌ Not using requirements.txt

→ Others can't run your project
