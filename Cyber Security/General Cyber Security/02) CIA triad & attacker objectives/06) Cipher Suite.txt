# What is a Cipher Suite? (Very Important for HTTPS & TLS)

A cipher suite is a set of cryptographic algorithms used together to secure a communication session in TLS/SSL (used in HTTPS).

Think of it like a recipe that tells your browser and the server:

> “Which encryption methods, hashing methods, and key-exchange methods should we use for this secure session?”

A cipher suite defines four major components:

---

# 1. Key Exchange Algorithm

### Purpose: How do we securely share keys?

Examples:

 ECDHE (Elliptic Curve Diffie-Hellman Ephemeral) → modern
 DHE
 RSA (old)

This defines how the browser and server agree on a shared secret.

---

# 2. Authentication / Digital Signature Algorithm

### Purpose: How do we verify the server’s identity?

Examples:

 RSA
 ECDSA

This is used in the server certificate.

---

# 3. Symmetric Encryption Algorithm

### Purpose: How is data encrypted once the handshake is complete?

Examples:

 AES-128-GCM
 AES-256-GCM
 CHACHA20-POLY1305

This is how actual data is encrypted after the handshake.

---

# 4. Hashing Algorithm (MAC / Integrity)

### Purpose: How do we ensure data wasn’t modified?

Examples:

 SHA-256
 SHA-384

Used for message authentication (MAC) or inside AEAD ciphers.

---

# Putting It All Together — Example Cipher Suite

### Example:

```
TLS_ECDHE_RSA_WITH_AES_256_GCM_SHA384
```

Let’s decode this:

| Part            | Meaning                                      |
| --------------- | -------------------------------------------- |
| ECDHE       | Key exchange – generates shared key securely |
| RSA         | Authentication – server certificate uses RSA |
| AES-256-GCM | Symmetric encryption for data                |
| SHA384      | Hash algorithm for integrity                 |

This suite is secure and modern.

---

# Another modern example

```
TLS_AES_128_GCM_SHA256
```

This is used in TLS 1.3, which simplifies cipher suites.

Why simplified?

→ Because in TLS 1.3,

 key exchange (always ECDHE),
 authentication,
  are standardized and NOT listed in the cipher suite.

---

# Why Are Cipher Suites Important? (Attacker View)

Attackers look for:

### 1. Weak cipher suites

Examples:

 RC4
 3DES
 NULL (no encryption)
 EXPORT-grade ciphers
 RSA Key Exchange (old)

### 2. Misconfigurations

 Server allowing outdated TLS versions (TLS 1.0/1.1)
 Weak hashing (MD5, SHA1)
 No forward secrecy (RSA key exchange)

Weak suites allow:

 MITM attacks
 Downgrade attacks
 Decrypting captured traffic

---

# TLS 1.3 Cipher Suites (IMPORTANT)

TLS 1.3 removed old stuff.

Only modern ones remain:

 `TLS_AES_128_GCM_SHA256`
 `TLS_AES_256_GCM_SHA384`
 `TLS_CHACHA20_POLY1305_SHA256`

Advantages:

✔ Always uses ECDHE → forward secrecy
✔ Faster
✔ Stronger algorithms only
✔ Smaller handshake

---

# Summary

A cipher suite specifies:

1. Key exchange (ECDHE)
2. Authentication (RSA/ECDSA)
3. Symmetric encryption (AES/ChaCha20)
4. Hashing (SHA256/SHA384)

It is the blueprint that defines how HTTPS secures your connection.
