Q9. What is the difference between NaN == NaN and Object.is(NaN, NaN)?

=> NaN is special:

In JavaScript, NaN stands for “Not-a-Number.”

By definition, NaN is not equal to anything, including itself.

== and === behavior

Both NaN == NaN and NaN === NaN return false.

This is because IEEE 754 floating-point standard defines 
NaN !== NaN.

Object.is() behavior

Object.is() is a more precise equality check.

It returns true for NaN compared to NaN, unlike == or ===.

Other differences

Check	Result
NaN == NaN	false
NaN === NaN	false
Object.is(NaN, NaN)	true
0 === -0	true
Object.is(0, -0)	false

Key Concept:

Use Object.is() when you want to detect NaN reliably or 
distinguish +0 and -0.

Example:

console.log(NaN == NaN);       // false
console.log(NaN === NaN);      // false
console.log(Object.is(NaN, NaN)); // true
