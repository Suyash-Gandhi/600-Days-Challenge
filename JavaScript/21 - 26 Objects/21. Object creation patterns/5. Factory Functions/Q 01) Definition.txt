A factory function is a regular JavaScript function that creates and returns an object.

Instead of using:

 `new` keyword
 Constructor functions
 Classes

You simply return an object manually.

```javascript

  return {
    name,
    age,

    }
  };
```

Here:

 `createUser` is the factory function
 Every time you call it, it creates a new object
 No `new` keyword is used

## 5ï¸âƒ£ Advantages of Factory Functions

1. Cleaner syntax
2. No accidental `new` mistakes
3. Easier to use closures
4. Better control over private variables

Example with private data:

```javascript
const createCounter = () => {
  let count = 0;

  return {
    increment() {
      count++;
    },
    getCount() {
      return count;
    }
  };
};

const counter = createCounter();
counter.increment();
console.log(counter.getCount()); // 1
```

Here `count` is private â€” cannot be accessed directly.

## 7ï¸âƒ£ One Important Note (Performance)

Factory functions create new copies of methods for every object.

Constructor functions + prototypes share methods.

So if you're creating thousands of objects, constructors or classes may be more memory-efficient.

## Final Definition (Exam Ready)

> A factory function is a JavaScript function that creates and returns an object without using the `new` keyword. It is a flexible and simple object creation pattern that allows encapsulation and private variables using closures.
